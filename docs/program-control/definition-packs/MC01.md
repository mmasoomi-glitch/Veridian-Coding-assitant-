# MC01 — Phone Home: Focus Now (real state)

- **Package/owner:** MC01 / veridian-develop
- **Feature:** A mobile-first "Home" tab showing REAL current state, replacing the legacy
  HUD that displays "unknown/empty" values on the phone.
- **Current problem:** On the phone the first screen is the legacy "Workspace Memory" HUD with
  ACTIVE PROJECT —, DESKTOP INDEX unknown, RAM/UPTIME blank, ACTIVE: UNKNOWN. Evidence:
  docs/program-control/verification/android-baseline/launch-result.md + first-screen.png.
- **Who is harmed / pain removed:** the owner opens the phone and learns nothing actionable;
  must go to the desktop. MC01 makes "where was I / what's active / what needs me" visible at a glance.
- **Desired visible phone outcome:** a Home tab (first tab) with: Active project (gitRepo@branch),
  active app + window title, modified-file count, latest commit, clipboard-has-secret flag (boolean
  only), and a "Waiting on you" list — all from real endpoints, with honest loading/empty/unavailable states.
- **Current code paths:** entry `src/main.tsx` → `LoginGate` → `TabbedApp` (tabs array). Data:
  `GET /api/telemetry/current` → `{ currentState:{ virtualDesktop, activeApp, windowTitle,
  workspacePath, gitRepo, gitBranch, latestCommit, modifiedFiles[], clipboardIsSecret,
  browserTitle }, timeline[] }`; `GET /api/waiting` → array of waiting items (defensive render).
- **Runtime truth today:** telemetry endpoint = LIVE (real PowerShell collector). Legacy HUD = stale/unknown.
- **Architecture/contracts to preserve:** TabbedApp tab registration pattern; `apiBase` + `credentials:"include"`
  fetch convention; dark-cockpit Tailwind; `motion/react` gentle fades (owner hates flicker); honest "unknown".
- **Reuse:** existing fetch pattern in TabbedApp/components; lucide-react icons; no new deps.
- **Forbidden changes:** do NOT touch server.ts auth, vault, routing, other tabs' logic, or any
  *.ps1; do NOT display secret VALUES (only the clipboardIsSecret boolean); no new dependency.
- **Security/privacy:** L0 read-only display. No secret values, no raw clipboard, no absolute paths
  shown (show gitRepo basename + branch, not workspacePath). data sensitivity: low (already-shaped telemetry).
- **Blast radius:** LEVEL 0 (read-only UI). 
- **Failure impact:** none beyond a non-rendering tab; legacy tabs unaffected.
- **Acceptance criteria:** new "Home" tab is first + default; shows live activeApp/window, gitRepo@branch,
  modified count, latest commit, clipboard-secret flag, and waiting list; shows a clear "telemetry
  unavailable" state when the endpoint fails (no fake values); no flicker on poll; tsc clean.
- **Negative cases:** endpoint 5xx/timeout → "unavailable" (not blank pretending live); empty waiting
  → "nothing waiting"; missing fields → "unknown" (never fabricated); secret value never rendered.
- **Test plan:** a tsx unit test for a pure `summarizeFocus(currentState)` helper (formats fields,
  never returns a secret value, marks unknowns) + tsc.
- **Phone runtime verification:** build APK → adb install -r → launch → screenshot shows the Home tab
  with real values (not "unknown" everywhere) when telemetry is reachable via adb reverse.
- **Rollback:** revert the new component + the one TabbedApp tab line; pure addition.
- **Exclusive files:** `src/components/FocusNow.tsx` (new), `src/components/focus-summary.ts` (new pure helper),
  `tests/mc01-focus.test.ts` (new), and the tabs array in `src/components/TabbedApp.tsx` (one entry + one render line).
- **Branch/worktree:** wp/mc01-home (writer worktree).
- **Independent reviewer:** Opus reviewer (not the author) + phone runtime verifier.
