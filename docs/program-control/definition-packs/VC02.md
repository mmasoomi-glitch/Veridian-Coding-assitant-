# VC02 / MC04 — Context Engine: current project + "where was I" (from REAL existing data)

- **Package ID:** VC02 (a.k.a. MC04)  ·  **Project boundary:** Veridian only.
- **Business purpose / intent:** V-INTENT-01 + V-INTENT-02 — at a glance tell the owner where
  they were, what project is active, and what changed, from REAL data (not the unknown-value HUD).
- **Owner pain removed:** opens desktop/phone and immediately knows the active project, branch,
  what's dirty/at-risk, the last "was doing / next step", and what's waiting — without hunting.
- **Developer pain removed:** one composite truth source instead of stitching 5 endpoints per surface.
- **Affected journey/surface:** desktop Home (FocusNow) + phone (same web build) + a new read endpoint.
- **Current observed behavior:** legacy HUD shows ACTIVE PROJECT/DESKTOP/RAM as "unknown/…"; MC01
  FocusNow shows telemetry+waiting but NOT the desktop brief, repo risk, or recent changes.
- **Evidence (file:line):** server.ts /api/telemetry/current (shapeTelemetry ~341-440); telemetry/
  persist.ts (workspace-sessions.json timeline cap 60); autopilot/desktop-briefs.ts (wasDoing/
  nextStep per "Desktop N"); telemetry/watcher.ts /api/waiting; orchestrator/repo-registry.ts
  scanReposSafe + classifyRisk; src/components/FocusNow.tsx + focus-summary.ts. (Per audit 2.)
- **Runtime classification:** telemetry/repo/waiting = LIVE; desktop-brief = LIVE-but-stale (only
  refreshed via /api/ai/summarize, which needs a provider); AI-summary text = UNAVAILABLE if no provider.
- **Reuse (no new collectors):** the above endpoints/stores. AI-brief is OPTIONAL/degraded-honest.
- **Exact files to modify:**
  - NEW `lib/context-engine.ts` — pure-ish builder `buildContextSnapshot()` that reads the existing
    stores (telemetry shape, desktop-briefs.json for the current virtualDesktop, /api/waiting items,
    repo risk via scanReposSafe, recent timeline) and returns a typed ContextSnapshot. NO secret
    values, NO absolute paths (repo name + branch only; clipboard = boolean only).
  - NEW server route `GET /api/context/current` in server.ts (open, read-only) → buildContextSnapshot().
  - MODIFY `src/components/FocusNow.tsx` (MINIMAL) — additionally fetch `/api/context/current` and
    render: "You were: <wasDoing> · Next: <nextStep>" (from the desktop brief, if present), top repo
    risk chip, and a "recent changes" line (last 3 timeline events). Keep existing telemetry+waiting.
    Honest states: brief missing → "no saved brief for this desktop"; AI text absent → omit (no fake).
  - NEW `tests/vc02-context.test.ts` — pure tests for buildContextSnapshot mapping + no-leak.
- **Forbidden changes:** do NOT touch auth/vault/routing/other tabs; no *.ps1; no new dependency; do
  NOT display secret values, absolute paths, or fabricate AI text when no provider is configured.
- **Data sensitivity / privacy boundary:** L0 read-only display; reuse existing redaction (clipboard
  boolean only, repo basename, no workspacePath). No OTP/keystroke/screen/log.
- **Blast-radius level:** LEVEL 0 (read-only endpoint + display).
- **Failure impact:** a non-rendering section; other tabs unaffected. **Rollback:** revert the new
  module + route + the FocusNow additions (pure addition).
- **Acceptance criteria:** /api/context/current returns real {telemetry, repo, brief|null, waiting,
  recentEvents} with NO secret/path; FocusNow shows the brief "was doing / next" when present and an
  honest "no saved brief" otherwise; recent-changes line shows real timeline; tsc clean; no flicker.
- **Negative cases:** endpoint 5xx → "unavailable"; empty desktop-brief → honest empty; no AI provider
  → omit AI text (never fake); missing fields → "unknown"; secret never rendered; path never rendered.
- **Required tests / verification:** unit (buildContextSnapshot: mapping + no absolute path + clipboard
  boolean); runtime API proof (curl /api/context/current on :3000); desktop browser proof (Home shows
  brief + risk); phone proof via adb (when connected). 
- **Branch/worktree:** wp/vc02-context (writer worktree).  **Exclusive ownership:** the 3 files above.
- **Big-LLM model:** cohere/north-mini-code:free (via scripts/ai/openrouter_bigllm_bundle.py).
- **Independent reviewer:** Opus reviewer (not author) + runtime verifier.
- **Owner decision required:** none to build L0. (Separate: enable an AI provider to get brief TEXT —
  optional; the engine works without it. Recorded for an owner-decision-pack if desired.)
